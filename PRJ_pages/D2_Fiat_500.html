ELECTRIC CAR CLUSTER & BMS

Customized Cluster for a vintage car

Full featured car cluster realization for a vintage car (1960' Fiat 500) retrofitted with electric powertrain.

BMS functionalities and logic is added to manage battery functions, such as range estimation, recharge detection, battery charge level prediction, and more.

details:

- UI design according to customer specifications on a round display, and its implementation on commercial HW
- Data collection and setting on MCU (Electric motor electronic control unit) via dedicated CAN bus
- External signals acquisitions (low/high beam lights, turn indicators, handbrake, RPM phonic wheel pick up detection)
- Real time data display on cluster, with indication of: vehicle speed, odometer, battery pack voltage and current level, battery charge level, gear, low/high beam, turn indicator, hanbrake
- Implementation of battery manager strategies (BMS) and functionalities for battery charge level prediction and indication, and battery recharge features
- Battery pack Over-Temperature / Over-Voltage protection, failure detection
- Additional UI design provided, inspired to Electric Hummer stock cluster

Phase 1: Idea & Research
The concept of the cluster design, initially took inspiration from the GMC Electric Hummer.
A feasibilty study was necessary to check the integration of the development board to the drivetrain control unit.
So an analisys of the drivetrain MCU (electric motor control unit) communication protocol was performed, including some preliminary tests aimed to acquire data / set commands, using a Vector Canalyzer.
Phase 2 - Design & UX
PC side development of a basic preliminary SW for the basic functions, including:

• Canonical cluster indicators:
> UI with custom gadgets for various indicators
> Speed, odometer, gear shift, turn indicator, high/low light beam, handbrake
> Electric interface development to acquire analog signals from vehicle

• BMS functions:
> Battery voltage level, battery status, battery current absorption
> Battery pack OverTemperature / OverVoltage protections
> Mode selection commands for electric motor (Power mode/ECO mode)
> implementation of logic for range estimation

A Demo mode was implemented as well, for an animated presentation of the UI to the customer.
Phase 3 - Prototyping
This phase involves the porting of the SW on Target board, adapting the UI to the circular display.
It was also the opportunity for a Boot time optimization, obtained overclocking the device, and disabling unused O.S. services.
Phase 4 - Testing
In Car integration and testing of the functionalities.
BMS parameter tuning and optimization for odomenter, speed and range estimation.

Vehicle speed calculation is made via electric motor rpm detection, or using a pick-up installed on the gear shift axle, by means of conditioning electronic and pulse counting.
Odometer is calculated via a time-integration of the vehicle speed, using precise timer library, so to to obtain a precision estimated around 0.8%.
